# 실사용 UX 리뷰

## 목표 사용자

연구실 행정 담당자 또는 출장 정산을 취합하는 구성원이 주 사용자입니다. 일반 구성원은 자기 파일을 올리는 것만 하면 되고, 관리자는 검토필요 문서만 빠르게 정리하는 구조를 목표로 합니다.

## 핵심 화면

### 1. 출장 건 목록

출장 건별 출장자 수, 문서 수, 검토필요 수, 미지정 수를 바로 보여줍니다. 관리자는 어떤 출장 건이 아직 정리되지 않았는지 먼저 판단할 수 있습니다.

### 2. 출장 상세 화면

상단에는 전체 진행률, 검토필요, 사람 미지정, 텍스트 미추출 수를 카드로 보여줍니다. 이어서 명단 병합, 사람 직접 추가, 문서 업로드, 출력, 검토 큐가 한 화면에 배치됩니다.

### 3. 검토 큐

`검토 필요`, `자동매칭`, `검토완료`, `사람 미지정`, 문서 유형별 필터를 제공합니다. 각 문서 행에는 파일명, 분류 신뢰도, 사람 매칭 신뢰도, 날짜/금액 힌트, 탐지된 이름, 추출 텍스트 미리보기가 표시됩니다.

## 중요한 UX 결정

완전 자동 확정은 위험합니다. 항공 예약서나 호텔 예약서에는 여러 사람이 함께 적히는 경우가 있고, 사진 영수증은 이름이 없는 경우가 많습니다. 그래서 시스템은 확신도가 높지 않거나 여러 사람이 탐지되면 `검토필요`로 남깁니다.
